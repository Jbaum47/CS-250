How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

  I would try and translate the users needs into a goal or feature of the program. These goals should be clear and easily defined so that they can be tested within the program. 
  User Stories are a great way to do this. They turn user needs and create testable features and cases which we can use to implement the desired features and develop them.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

  I think being communicative and open to working together is a great way to approach developing. Most Agile processes involve this open communication. 
  I like the idea of short daily meetings, like the daily Scrum, that make sure everyone is on the same page and working together.

What does it mean to be a good team member in software development?

  I think being a good team member is being someone who does their best at their work and is open to criticism. Being able to talk openly as a team without having to worry 
  too much about everyones feelings is great for development. With this, we can focus on creating the best software possible as a team.
